# Project Playbook

Last updated: 2026-05-24

## Launch Flow

1. Archive official sources.
2. Write specs and claim boundaries.
3. Push specs to GitHub.
4. Bootstrap Next.js site.
5. Implement core route/data/component system.
6. Add Gumroad-ready product files and skill pack.
7. Generate raster visual assets.
8. Build and QA locally.
9. Deploy to Vercel.
10. Add custom domain after user supplies it.
11. Add Gumroad links after user publishes products.
12. Submit indexing tasks after production domain is stable.

## Credential Rules

- Do not commit `.env` files.
- Do not paste tokens into docs.
- Read deployment credentials from local environment only.

## Deployment Placeholder

Until the final domain is supplied, use a Vercel deployment URL. Production custom-domain verification is deferred.

## Residual Risks

- Official docs can change quickly.
- The current content is based on official docs checked on 2026-05-24.
- Gumroad buyer flow cannot be fully checked until product URLs exist.
- Custom domain cannot be verified until the user supplies the domain and DNS provider is confirmed.
